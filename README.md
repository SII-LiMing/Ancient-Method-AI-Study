## 💡 为什么要有这个项目？ (The Manifesto)

我们正处于一个被称为 **Vibe coding** 的狂飙时代。
在这个时代，你只需要对大语言模型随意输入一句 *"帮我写个带 RoPE 的 GQA Transformer"*, 几秒钟后，一份完美封装的代码就会出现在你的屏幕上。只要跑通了，没人再去关心底层发生了什么。

**但代价是什么？**
代价是我们正在失去对张量维度的敬畏，对梯度流动的感知，以及对底层数学公式的共情。当 Bug 出现时，开发者沦为了只会把报错复制回给 AI 的“中转站”。

**Ancient-Method-AI-Study** 是一次逆行。
本项目属于那些渴望真正理解 AI 引擎运转原理的初学者与信徒。我们主张 **「古法编程」**：

1. 🧱 **拒绝黑盒，直击本质 (Reject Black Boxes)**
   不要理所当然地调用现代框架中那些被高度封装的魔法接口。我们将层层剥开这些精致的包装，从最基础的计算图拓扑出发，去手写、去拆解每一个核心环节。我们致力于对 AI 进行一次彻底的“技术祛魅”。

2. 📐 **死磕张量，敬畏维度 (Reverence for Dimensions)**
   在多维矩阵的流转中，形状（Shape）即是语义。我们绝不依赖 AI 替我们用试错的方式去缝合代码。无论是何种复杂的网络架构，我们要求对每一笔数据在内存中的物理切分、流动与重组，都做到绝对掌控、了如指掌。

3. 📜 **原典教义，公式直译 (Mathematical Originalism)**
   摒弃那些经过层层迭代、为了工程妥协而面目全非的现代“魔改”库，重新回到开山之作的原始数学定义。我们将严丝合缝地把论文里的每一行推导、每一个符号，直译为最纯粹的代码。这里只有从数学到程序的绝对映射。

4. 🔍 **全栈打通，事无巨细 (Full-Stack Granularity)**
   这不是一个浅尝辄止的“调包”教程，而是一场事无巨细的苦行。任何在成熟框架中看似理所应当的经验常数、缩放因子、或是信息路由逻辑，在这里都会被彻底打碎、揉开，去穷究其背后的“为什么”。
